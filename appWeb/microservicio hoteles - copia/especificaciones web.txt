Microservicio de Gestión de Hoteles (Node.js/Express):
Este servicio ya lo tienes implementado y se encarga de la gestión de la información de los hoteles.

Funcionalidades existentes:

Listar hoteles.
Obtener detalles de un hotel específico.
Añadir y actualizar información de hoteles.
Funcionalidades a añadir:

Interfaz de Reseñas: Podrías añadir endpoints para obtener las reseñas asociadas con cada hotel, que serán recuperadas del segundo microservicio a través de llamadas HTTP.
Comunicación con el Microservicio de Reseñas: Para mantener la consistencia de los datos entre servicios, cada vez que se añada o actualice un hotel, podría notificarse al servicio de reseñas para que actualice su base de datos si es necesario.
Microservicio de Gestión de Reseñas y Usuarios (FastAPI/MySQL):
Este nuevo servicio manejará todo lo relacionado con usuarios y sus reseñas.

Funcionalidades:

Autenticación de Usuarios:

Registro de usuarios con validación de datos.
Inicio de sesión con generación de tokens (JWT podría ser una buena opción).
Gestión de sesiones y seguridad (por ejemplo, hash de contraseñas).
Gestión de Reseñas:

Publicar una nueva reseña asociada a un hotel y un usuario.
Listar todas las reseñas de un hotel específico.
Actualizar y eliminar reseñas (esto debe ser restringido solo al usuario que creó la reseña o a administradores).
Interfaz de Hoteles:

Aunque la información principal del hotel se maneja en el otro microservicio, este servicio necesita tener una referencia a los hoteles para poder asociar reseñas. Esto podría ser un ID de hotel con un mecanismo para verificar la existencia del hotel en el otro microservicio.
Base de Datos MySQL:

Una tabla para usuarios con la información necesaria para el login y el registro.
Una tabla para reseñas que incluya el ID de usuario, el ID de hotel, la puntuación numérica y el texto de la reseña.

Microservicio de Gestión de Hoteles (MongoDB):
La base de datos de este microservicio almacenará todo lo relacionado con la información de los hoteles.

Colección de Hoteles:
ID del Hotel: Un identificador único para cada hotel.
Nombre: El nombre del hotel.
Descripción: Una descripción del hotel, incluyendo servicios y comodidades.
Dirección: Dirección completa del hotel.
Estrellas: Calificación del hotel en términos de estrellas.
Ubicación: Datos geográficos como el código postal, el municipio, y el código de territorio.
Imágenes: URLs a imágenes del hotel.
Datos de Contacto: Número de teléfono, email, etc.
Precio: Rango de precios o precios por tipo de habitación.
Cantidad de Habitaciones: Número total de habitaciones disponibles.
Microservicio de Gestión de Reseñas y Usuarios (MySQL):
La base de datos de este microservicio contendrá información sobre los usuarios y sus reseñas de hoteles.

Tabla de Usuarios:

ID del Usuario: Un identificador único para cada usuario.
Nombre de Usuario: El nombre de usuario elegido por la persona.
Correo Electrónico: Dirección de correo electrónico del usuario.
Contraseña: Contraseña del usuario (almacenada como un hash seguro).
Información de Perfil: Datos adicionales como nombre completo, fecha de registro, etc.
Tabla de Reseñas:

ID de la Reseña: Un identificador único para cada reseña.
ID del Hotel: El identificador del hotel al que se refiere la reseña (debe coincidir con un ID en la base de datos de hoteles).
ID del Usuario: El identificador del usuario que deja la reseña.
Puntuación: Calificación otorgada al hotel, generalmente de 1 a 5.
Texto de la Reseña: El comentario o texto de la reseña.
Fecha de Creación: La fecha y hora en que se creó la reseña.
